Start here

FNLLA Web is the stable no-build runtime layer for TechAyo websites. This page is the routing map for the docs set, while the deeper pages hold the detailed implementation rules.

Stable runtime contract

<link rel="stylesheet" href="assets/css/fnlla-web.css">
<script src="assets/js/fnlla-web.js"></script>

The public runtime starts with one stylesheet, one script and the local assets/icons/ bundle. The rest of the docs explain how to structure markup, choose components and keep that contract stable.

Choose your working mode

  • Consumer mode: copy dist/fnlla-web/ or work directly from assets/, then build a website on top of it.
  • Maintainer mode: change src/, republish runtime files and run validation before release.
  • Default rule: if a change only serves one website, keep it in the website project instead of pushing it into the framework.

Stable authoring rules

These are the rules worth keeping in your head while reading the rest of the docs set.

Layout first, components second

Start from wrapper, section, container and grid, then place cards, forms, stats or sections inside that shell. Do not let cards become the default page skeleton.

Tokens before selector overrides

Rebrand through shared variables such as --fnlla-color-primary, --fnlla-space-* and typography tokens before editing component selectors or runtime CSS directly.

Hooks stay on the documented path

Use data-fnlla-* hooks, real aria-controls targets and window.FNLLAWEB.init(root) for dynamic DOM insertion. Legacy data-ui-* patterns are outside the stable contract.

Icons stay local

Use the shipped local bundle under assets/icons/ or assets/icons/sprite.svg. Keep visible text for meaning and use icons as reinforcement, not as the only label.
