Layout responsibility map

Use this page as the structural source of truth: wrapper owns the page shell, section owns the full-width band, container owns the inner line and grid owns columns inside that line.

Outside-to-inside recipe

Shell first

  • wrapper keeps one vertical page flow.
  • header, main and footer stay semantic.
  • section controls spacing, background and rhythm.
  • container keeps a shared readable page line.
  • grid is where column decisions happen.

Decision order

Decide shell, then band, then width, then columns. Components such as cards, stats or forms come after that structure is already stable.

<div class="wrapper">
  <header>...</header>
  <main>
    <section class="section">
      <div class="container">
        <div class="grid grid-2 gap-md">...</div>
      </div>
    </section>
  </main>
  <footer>...</footer>
</div>

Section and container contract

Container line

Section owns the band

Keep backgrounds and vertical rhythm on the outer section, then align content inside one shared container.

<section class="section section-alt">
  <div class="container">
    <div class="card">...</div>
  </div>
</section>

Choose the width

Container size is the first local layout choice after the shell. Pick width based on reading comfort and content density, not on how much space happens to be available.

Container width scale

container-sm

Use for tighter reading widths, quieter single-column copy and support pages.

container

Use as the default page line for most marketing, docs and service layouts.

container-lg

Use when comparisons, data or media need more horizontal room.

<div class="container-sm">...</div>
<div class="container">...</div>
<div class="container-lg">...</div>

Choose the grid

Grid classes describe column count only. They do not decide what the inner blocks are, so the same structure can hold cards, forms, stats or plain content.

1-column grid

Single column

</div>

2-column grid

Column one

Anti-patterns to avoid

Most layout drift in downstream projects comes from responsibility leaks. These are the mistakes worth catching early in review.

Do not use cards as the page shell

Cards are content surfaces. They should sit inside a stable shell, not replace wrapper, section or container as the default page skeleton.

Do not nest containers without a reason

Unnecessary nested containers create drifting inner lines and unpredictable rhythm. Use one container per band until a genuine sub-layout needs otherwise.

Do not use wide multi-column grids for long prose

Long-form copy belongs in calmer widths. Wide three- or four-column grids are for grouped summaries, not for dense reading blocks.

Do not skip section > container

Dropping straight into ad hoc widths makes background bands, spacing and alignment harder to reason about. Keep the section-band and container-line split intact.
